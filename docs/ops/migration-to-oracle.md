# musclead 移行手順書（AWS → Oracle 無料VM）

## 全体像
```
            ┌─────────────── Oracle 無料VM（1台・$0）───────────────┐
ユーザー ──▶│  Caddy(TLS) ──▶ Go(アプリ) ──▶ MySQL(DB)             │
            │                         └─ relay(メール送信ポーリング) │
            └──────────────┬───────────────────┬────────────────────┘
                           │                   │
                   画像 ▶ Cloudflare R2   メール ▶ Resend
                   DNS: Cloudflare（api.musclead.com → VMのIP）
```
- VMに載せる: Go / MySQL / Caddy
- 外部の無料サービス: 画像=R2、メール=Resend、DNS=Cloudflare
- フロント(Vercel)は変更なし

---

## 事前に作る無料アカウント（あなた）
1. **Oracle Cloud**（VM用）
2. **Cloudflare**（R2＝画像保存／DNS）
3. **Resend**（メール送信）

---

## フェーズ A: コード側の差し替え　【主に私】
AWS固有の部分を、VM/外部サービス向けに変える。
- [ ] 画像: S3クライアントの接続先を **R2** に（環境変数でエンドポイント切替。コードはほぼ流用）
- [ ] メール: SES送信 → **Resend のAPI送信** に置換
- [ ] 非同期: SQS/Lambda/DynamoDB をやめ、**MySQLの outbox テーブル＋アプリ内ポーリング**に一本化（重複防止もMySQLの表で）
- [ ] 設定: SSM → **.env ファイル**で渡す
- [ ] 動作確認: ローカルの docker-compose で「画像アップ→表示」「支払い→メール」が通るか
> ここは Go のコード変更。あなたの学習方針に合わせ、私が書く/ヒントどちらでも。

## フェーズ B: Oracle VM を作る　【あなた、私が画面ごと案内】
- [ ] Oracle Cloud にログイン → **Compute → Instances → Create**
- [ ] イメージ: **Ubuntu**、シェイプ: **Ampere A1（Always Free）** 1〜2 OCPU / 6〜12GB
- [ ] SSHキーを登録（ログイン用）
- [ ] 作成後の **公開IP** をメモ
- ⚠️ **つまずきポイント**: A1は「在庫切れ」で作れない時がある→数回リトライ/別AZ。作れたら勝ち。

## フェーズ C: VMをセットアップ　【あなた、コマンドは私が用意】
SSHでVMに入って:
- [ ] **Docker** を入れる（コマンド1行スクリプトを私が渡す）
- [ ] **80/443ポートを開ける**（⚠️Oracleは2か所必要: ①管理画面のSecurity List ②VM内のファイアウォール。手順は私が用意）
- [ ] アプリ一式を置く（`git clone` でこのリポジトリを取得）

## フェーズ C2: 全部起動（docker-compose）　【私が設定ファイル作成】
- [ ] `docker-compose.yml`（**Go＋MySQL＋Caddy**）と `.env`、`Caddyfile` を用意（私が作る）
- [ ] `docker compose up -d --build` で起動
- [ ] MySQLデータは **永続ボリューム**に保存（再起動で消えない設定）
- [ ] Caddyが `api.musclead.com` のTLS証明書を自動取得

## フェーズ D: データ移行　【私が手伝う】
- [ ] **DB**: 現RDSから `mysqldump` で書き出し → VMのMySQLへ取り込み（踏み台経由。私が手順実行）
- [ ] **画像**: 現S3 → R2 へコピー（`rclone` で一括。私が実行）

## フェーズ E: 切り替え（DNS）　【あなた、私が確認】
- [ ] `api.musclead.com` の向き先を **VMの公開IP** に変更（A レコード）
- [ ] `app.musclead.com`(Vercel)は**そのまま**
- [ ] 反映後、本番で動作確認（ログイン／画像／支払い）

## フェーズ F: バックアップを仕込む　【私が用意】
- [ ] 毎日1回 `mysqldump` → **R2へ自動退避**する cron（VMが消えても復元できる保険）

## フェーズ G: AWSを止める　【あなた、私が指示】
- [ ] 数日問題なければ、AWS側を停止して課金ゼロに（`terraform destroy` 等。消す順番は私が指示）
- [ ] ドメインだけは維持（年~$15）

---

## ざっくり所要
- A(コード) 1〜2日 / B〜C(VM構築) 半日 / D(データ) 半日 / E〜F 半日 / G 後日
- 合計 **2〜3日**（実作業）。慌てず1フェーズずつ。

## 安全策
- **切り替えはDNSだけ**なので、問題あれば向き先をAWSに戻すだけで即ロールバック可能。
- AWSは**最後まで消さない**（Gは一番最後）。並走させて安全に乗り換える。
